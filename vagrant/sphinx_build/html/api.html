API Reference¶

Get list of restaurants¶

GET /api/restaurants¶

Get a list of restaurant records in the database.

Authentication:

Not required.
Response:

JSON
Example:
{
  "restaurants": [
    {
      "created_by": 1,
      "id": 1,
      "name": "Urban Burger",
      "note": "",
      "phone": "555-1234"
    },
  ]
}

Get restaurant menu¶

GET /api/menu¶

Get a list of restaurant menu items from the database. Enter either a restaurant name or it’s database ID.

Authentication:

Not required.
Parameters:
  • restaurant_id (optional) – Database ID for a restaurant.
  • restaurant (optional) – Name of a restaurant in the database.
Response:

JSON
Example:
{
  "menu": [
    {
      "bad_count": 0,
      "course": "Entree",
      "created_by": 1,
      "description": "a Vietnamese noodle soup.",
      "favorite_count": 0,
      "good_count": 0,
      "id": 16,
      "name": "Pho",
      "price": "8.99",
      "restaurant_id": 3,
      "restaurant_name": "Panda Garden"
    },
  ]
}

Get list of favorites¶

GET /api/favorites¶

Gets a random list of a user’s favorited items. Pass a user_id as a parameter or default to the currently logged in user. Returns an error if neither is found. The default limit is three.

Authentication:

Optional.
Parameters:
  • user_id (optional) – ID of user.
  • limit (optional) – Max number of items to return.
Response:
